layout

在Android中，布局是构建界面的基础，选择合适的根布局可以简化布局过程并提高性能。下面是一些常用的根布局和它们的特点：

  1. ConstraintLayout（约束布局）：
    ConstraintLayout是目前使用频率最高的布局之一。它通过子元素与父元素以及子元素之间的约束关系来确定元素的位置和大小。这使得它能够灵活适配不同尺寸的屏幕，并且具有良好的性能。对于使用Android Studio 3.0及以上版本的开发者来说，ConstraintLayout是一个强大且推荐的选择。
  2. RelativeLayout（相对布局）：
    RelativeLayout允许通过子元素之间的相对关系来定义布局。你可以使用诸如layout_below、layout_alignParentRight等属性来指定子元素的位置。RelativeLayout的代码相对简洁，但是当嵌套层级较多时，性能可能会下降。
  3. LinearLayout（线性布局）：
    LinearLayout通过纵向或横向的线性方式来排列子元素。你可以使用weight和gravity属性来调整子元素的大小和对齐方式。LinearLayout的代码非常简单，适用于简单的布局需求。
  4. FrameLayout（帧布局）：
    FrameLayout是最简单的布局之一。子元素会堆叠在左上角，并且你可以通过gravity属性改变它们的对齐方式。FrameLayout适用于包含少量视图的简单布局需求。
  5. GridLayout（网格布局）：
    GridLayout以网格的形式来排列子元素。你可以通过rowCount和columnCount属性来调整网格的行数和列数，并使用layout_row和layout_column属性来指定子元素的位置。
  6. TableLayout（表格布局）：
    TableLayout以表格的形式来排列子元素，类似于HTML中的表格结构。每一行都是一个TableRow，适用于显示具有明确行与列的数据。
  7. ScrollView（滚动布局）：
    ScrollView是一个滚动布局，只能包含一个子元素。当内容超过屏幕可见范围时，用户可以通过滚动来查看剩余内容。ScrollView常用于显示较长的内容。

不同的根布局,代码可能不同

例如

<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
                android:layout_width="match_parent"
                android:layout_height="match_parent"> 
    <ImageButton
    android:id="@+id/imageButton"
    android:layout_width="60dp" 
    android:layout_height="60dp"
    android:layout_alignParentRight="true"
    android:layout_alignParentTop="true"
    app:srcCompat="@drawable/ic_cute_cat" />

</RelativeLayout>

同樣是右上角


<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical"
    tools:context=".MainActivity">

        <ImageButton
        android:id="@+id/imageButton"
        android:layout_width="60dp"
        android:layout_height="60dp"
        app:layout_constraintRight_toRightOf="parent"
        app:layout_constraintTop_toTopOf="parent"
        app:srcCompat="@drawable/ic_cute_cat" />
    

</androidx.constraintlayout.widget.ConstraintLayout>

TableLayout和GridLayout的区别。😊

TableLayout和GridLayout都是Android中的布局管理器，可以将容器划分为网格，每个网格可以放置一个组件。但是它们也有一些不同之处，主要有以下几点：

  • TableLayout不需要明确地声明包含多少行多少列，而GridLayout需要设置rowCount和columnCount属性来指定行数和列数。
  • TableLayout通过添加TableRow或其他组件来控制表格的行数和列数，而GridLayout通过设置layout_row和layout_column属性来指定组件所在的行和列。
  • TableLayout无法做出跨行跨列的效果，每行每列都是挨着的，就算是单元格设置Collapsed属性，如果后面有组件，也会填充上来。而GridLayout可以设置layout_rowSpan和layout_columnSpan属性来让组件横跨多少行或者纵跨多少列。
  • TableLayout可以设置单元格的格式为Shrinkable（可收缩），Stretchable（可拉伸），Collapsed（隐藏的）。而GridLayout可以设置layout_gravity属性来调整组件在单元格里的位置，以及设置layout_columnWeight和layout_rowWeight属性来设置各个组件的大小比例。

生命周期

当开发一个 Android 应用程序时，了解 Android 生命周期非常重要，因为它决定了应用程序的行为和状态管理。Android 提供了一组生命周期方法，使开发者能够在特定的时间点执行代码，以便适应应用程序的不同状态。下面是 Android 应用程序的主要生命周期方法及其顺序：

  1. onCreate():
    这是应用程序的第一个生命周期方法，当应用程序被创建时调用。在这个方法中，通常会进行一些初始化操作，例如设置布局和变量，以及绑定视图。

  2. onStart():
    在 onCreate() 方法之后，应用程序进入 onStart() 方法。在这个方法中，应用程序变得可见，但它还无法与用户进行交互。

  3. onResume():
    当应用程序进入前台并可与用户进行交互时，onResume() 方法被调用。在这个方法中，通常会注册监听器、开始动画或者恢复暂停的任务。

  4. onPause():
    当另一个 Activity 获得焦点或者用户按下 Home 键时，应用程序进入 onPause() 方法。在这个方法中，通常会暂停活动的任务、保存数据或释放资源。

  5. onStop():
    当应用程序完全不可见时，即进入后台运行或者被其他应用程序覆盖时，onStop() 方法被调用。在这个方法中，通常会释放占用的资源、保存用户数据或者停止动画效果。

  6. onRestart():
    如果应用程序从 onStop() 方法返回前台运行，onRestart() 方法将被调用。在这个方法中，通常会执行一些重新初始化的操作。

  7. onDestroy():
    当应用程序被销毁时，或者由于内存不足而被系统终止时，onDestroy() 方法被调用。在这个方法中，通常会释放所有的资源、取消注册的监听器或保存最终的用户数据。

除了这些核心的生命周期方法外，还有其他一些方法用于处理特定事件和状态的变化：

  • onRestoreInstanceState() 和 onSaveInstanceState():
    当活动由于配置更改（例如屏幕旋转）而被销毁和重新创建时，可以使用这两个方法来保存和恢复活动的状态。

  • onActivityResult():
    当一个活动启动另一个活动并等待返回结果时，可以使用 onActivityResult() 方法来处理返回的结果。

  • onConfigurationChanged():
    当设备配置发生更改（例如屏幕旋转或语言更改）时，onConfigurationChanged() 方法被调用。

了解并正确处理这些生命周期方法是开发 Android 应用程序的关键，以确保应用程序在不同的状态和事件下正常工作，并提供良好的用户体验。

当涉及到线程和数据资源的保存、阻塞和销毁时，以下情况需要考虑：

  1. 线程的保存和恢复：
    当应用程序在 onPause() 和 onResume() 方法之间切换时，线程可能需要被暂停和恢复。为了保存线程的状态，可以在 onPause() 方法中停止线程的执行，并在 onResume() 方法中重新启动线程。这可以通过使用标志变量或者线程的暂停和恢复方法来实现。

  2. 数据资源的保存和恢复：
    当应用程序处于后台或者被销毁时，需要保存关键的数据资源，以便在应用程序恢复或重新创建时恢复其状态。可以通过以下方法保存和恢复数据资源：

    • onSaveInstanceState() 和 onRestoreInstanceState() 方法：
      在活动被销毁和重新创建时，可以使用 onSaveInstanceState() 方法保存数据资源的状态，并在 onRestoreInstanceState() 方法中恢复状态。这些方法提供了一个 Bundle 对象，可以用于存储和检索数据。

    • 持久化存储：
      对于较大或需要长期保存的数据，可以使用持久化存储方法，例如将数据保存到数据库、文件系统或 SharedPreferences 中。这样即使应用程序被销毁，数据也可以被恢复。

  3. 阻塞和销毁：
    在某些情况下，可能需要阻塞或销毁线程以及释放占用的资源。以下是一些常见的情况：

    • 阻塞线程：
      当需要等待某个操作完成或等待某个条件满足时，可以使用线程的阻塞方法，如 Thread.sleep() 或者使用等待和通知机制（如 wait() 和 notify()）来阻塞线程的执行。

    • 销毁线程：
      当线程不再需要时，可以通过停止线程的执行来销毁它。这可以通过设置标志变量或调用线程的 interrupt() 方法来实现。

    • 释放资源：
      当应用程序不再需要某些资源时，应该及时释放它们，以避免资源泄漏。这包括关闭文件、数据库连接、网络连接或其他系统资源的释放。

综上所述，正确处理线程和数据资源的保存、阻塞和销毁是确保应用程序在各种状态下正常运行和提供良好用户体验的关键。通过合理地管理线程和数据资源，可以确保应用程序在各种情况下都能正确地保存和恢复状态，并有效地使用系统资源。
